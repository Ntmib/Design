# Воркфлоу: Презентация для трансляции (navigator.html)

> Стандарт оформления `navigator.html` для каждого YouTube-ролика. Это слайдовая презентация, которую Дмитрий транслирует зрителям во время записи: они видят буллиты, инсайты, цитаты и сравнительные таблицы. Технических подсказок ведущему здесь НЕТ — для этого есть `teleprompter.html`.

## Эталон-референс

**Канонический пример:** [`scripts/fayl-karpati-claude-md/navigator.html`](../scripts/fayl-karpati-claude-md/navigator.html)

Утверждено Дмитрием 2026-05-08 как стандарт для всех будущих роликов. Любую новую презентацию начинай с копирования стилей и компонентов оттуда — не изобретай заново.

## Цель формата

`navigator.html` это контент для зрителя, не шпаргалка для ведущего:

- Зритель видит структуру урока на экране
- Ключевые цифры, цитаты и кейсы — крупно, чтобы воспринимались с трансляции
- Сравнения «было/стало» — двумя колонками, без листания PDF
- Финальные инсайты — отдельными слайдами на полный экран

**Что НЕ должно попадать в презентацию:**

- Open loops для удержания зрителя («сейчас покажу...», «через 5 минут...»)
- Метки времени блоков ролика (1.5'', 4'') — это для съёмки, в `teleprompter.html`
- Метки переключения камера/экран
- Карточки «⚠ Адаптировать» с инструкциями ведущему
- Внутренние пометки для съёмки

Если в сценарии есть open loop или техническая подсказка — она остаётся в `scenario.md` и `teleprompter`, в `navigator.html` идёт только смысл.

## Типы слайдов и их шаблоны

### 1. Титульный слайд

Огромная цифра (43.000 / 65 / 8 фишек) + название ролика + подзаголовок + пиксельный тег `@neurobussines`.

```html
<section class="slide" data-slide="1">
  <div class="slide-tag">@NEUROBUSSINES</div>
  <div class="slide-inner text-center">
    <div class="hero-num">43.000</div>
    <div class="hero-num-suffix">установок за неделю</div>
    <div class="hero-title">Один файл, который меняет Claude Code</div>
    <div class="hero-subtitle">4 принципа от Андрея Карпати</div>
    <div class="hero-pixel-tag">CLAUDE.MD · 65 СТРОК</div>
  </div>
</section>
```

### 2. Слайд с буллитами

3-5 пунктов с пиксельными цифрами (01/02/03) или эмодзи в круге. На белой карточке с терракотовым бордером слева.

Используется для планов («Сегодня три вещи»), описаний автора, перечислений артефактов.

### 3. Слайд-принцип

Большая цифра принципа (Press Start 2P, 8rem) + название (Oswald 6rem) + tagline + терракотовая карточка с формулировкой «📝 ЗАПОМНИ».

```html
<div class="principle-hero">
  <div class="principle-num">01</div>
  <div class="principle-name">Думать перед<br>кодом</div>
  <div class="principle-tagline">Claude по умолчанию НЕ думает...</div>
</div>
<div class="principle-card">
  <strong>Сначала думай, потом пиши.</strong><br>
  Несколько интерпретаций → выбор → план → код.
</div>
```

### 4. Слайд-цитата (тёмный)

Класс `slide.dark`. Тёмная карточка с большим открывающим « + цитата (Oswald 3rem курсив) + автор моноширинным внизу справа. Самые сильные цитаты выделять `accent-light` на ключевых словах.

### 5. Слайд-кейс (двухколоночный)

Промт зрителя сверху → две колонки `case-side.bad` (красный бордер) и `case-side.good` (зелёный бордер). В каждой колонке: метка «❌ ЧТО ДЕЛАЕТ» / «✅ ЧТО ДОЛЖНО БЫТЬ» + headline + 4-6 буллитов.

Используется для всех демонстраций «было/стало», особенно для кейсов из источников типа `EXAMPLES.md`.

### 6. Слайд с командой (терминал)

Mac-style терминал с тремя точками + JetBrains Mono тело. Подсветка `cmd` (терракот), `arg` (голубой), `dim` (серый), `green` (✓ done).

Используется для команд установки, промтов, конфигов.

### 7. Сравнительная таблица

`compare-table` с тёмной шапкой. Метрика слева, варианты справа. Класс `bad`/`good`/`neutral` на ячейках для подсветки направления.

### 8. Акцентный слайд (терракотовый)

Класс `slide.accent` — фон-градиент терракот → терракот-тёмный. Используется для главных выводов или вступлений к секциям («3 способа поставить файл»).

### 9. Финальная цитата

Тёмный слайд + `final-quote-card` с гигантской открывающей кавычкой (Oswald 14rem) + цитата (Oswald 4.5rem) + атрибуция моноширинным внизу.

### 10. Слайд лид-магнита

3 `gift-item` с иконкой в терракотовом квадрате + название + описание. Снизу `gift-cta` тёмный блок с CTA-фразой.

## Технические правила

### Scroll-snap

```css
html { scroll-snap-type: y mandatory; }
.slide { min-height: 100vh; scroll-snap-align: start; scroll-snap-stop: always; }
```

Каждый слайд занимает ровно один экран. Нет «двух слайдов в одном» — если контент не помещается, делим на два слайда.

### Навигация

- **Прогресс-бар сверху** (`progress-track` + `progress-fill`) — терракотовый градиент, заполняется по мере прокрутки
- **Счётчик внизу справа** — пиксельный, формат «05 / 26»
- **Стрелки внизу слева** — круглые тёмные кнопки ← →
- **Хоткеи:** ↓ → Space PageDown — следующий, ↑ ← PageUp — предыдущий, Home / End — край, F — полный экран

### Метки слайда

Каждый слайд начинается с `slide-tag` в верхнем левом углу — пиксельный шрифт, типа `02 · АВТОР` или `КЕЙС ИЗ EXAMPLES.MD · DRIVE-BY REFACTORING`. Это для зрителя — он понимает где находится.

### Адаптация

Один media query на 1200px (одноколоночные кейсы), второй на 768px (мобильная урезанная типографика). Презентация оптимизирована под десктоп — на мобильных она просто не сломается.

### Inline-стили

**Запрещены полностью.** Если нужна разовая стилизация — добавляй utility-класс в `<style>` и используй его. IDE-линтер ловит inline.

## Типовая структура слайдов ролика

Объём презентации — 20-30 слайдов на ролик длиной 12-15 минут. Расклад на примере fayl-karpati-claude-md:

| Группа | Кол-во слайдов | Что включить |
|--------|---------------|--------------|
| Открытие | 3-4 | Титул + автор/контекст + план + ключевая цифра |
| Контент-блок 1 (например «4 принципа») | по 2-3 на пункт | Принцип-слайд + цитата (если есть) + кейс/пример |
| Контент-блок 2 (например «3 способа») | 4-5 | Вступление + по слайду на каждый способ + сравнение |
| Тест/демо | 2-3 | Постановка + результат + инсайт |
| Бонус | 1-2 | Универсальность / расширение применимости |
| Закрытие | 3 | Главный вывод + финальная цитата + CTA-лид-магнит |

Итого: 20-30 слайдов.

## Чек-лист перед сборкой нового navigator.html

- [ ] Скопировать стили из `scripts/fayl-karpati-claude-md/navigator.html` целиком (не переписывать дизайн-систему)
- [ ] Заменить только `<title>`, контент `<section>`-ов и `data-slide`
- [ ] Убрать все open loops, тайминги, метки камера/экран — это в teleprompter
- [ ] Каждая цитата — отдельный слайд, не блок внутри другого
- [ ] Каждый кейс «было/стало» — отдельный слайд с двумя колонками
- [ ] Главный вывод и финальная цитата — на полный экран, в конце
- [ ] Лид-магнит финальным слайдом, с тремя `gift-item` и `gift-cta`
- [ ] Никаких inline-стилей — только классы
- [ ] Открыть в браузере, прокликать ←/→/F и проверить ритм
- [ ] Закоммитить в репо вместе со сценарием

## Связь с другими файлами в папке ролика

```
scripts/{slug}/
├── transcript.txt          ← оригинал источника (если разбор чужого)
├── scenario/scenario.md    ← полная архитектура: open loops, тайминги, демо
├── teleprompter.md         ← разговорный текст для съёмки
├── teleprompter.html       ← HTML-телесуфлёр с автопрокруткой
└── navigator.html          ← ЭТА презентация для трансляции зрителям
```

Каждый файл — для своей задачи:

| Файл | Кому | Когда |
|------|------|-------|
| `scenario.md` | Дмитрию (планирование) | Перед съёмкой при подготовке |
| `teleprompter.md` | Дмитрию (читка) | Во время съёмки на втором экране |
| `teleprompter.html` | Дмитрию (автопрокрутка) | Альтернатива .md, удобнее визуально |
| `navigator.html` | **Зрителям** (трансляция) | Транслируется в OBS как источник |
