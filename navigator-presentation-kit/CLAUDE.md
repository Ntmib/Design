# CLAUDE.md — инструкции для AI-агентов

> Этот файл читают AI-агенты (Claude Code, Cursor, Copilot, Codex), когда работают в этом проекте. Он определяет стандарт презентаций для трансляции во время записи видео.

## Контекст проекта

Это **Navigator Presentation Kit** — переносной набор для создания HTML-слайдов в стиле «Крафтовый терминал». Презентации транслируются зрителям во время записи YouTube-видео или эфиров: ведущий показывает слайды как «контент на экране», а сам говорит по своему телесуфлёру.

**Главное:** `navigator.html` — это контент для зрителя, не шпаргалка для ведущего.

## Что у тебя есть

| Файл | Назначение |
|---|---|
| `navigator-template.html` | **Эталон-референс.** Копируй из него блок `<style>` целиком, не переписывай дизайн. Структура `<section>` — пример заполнения, замени на свою тему |
| `navigator-guide.md` | Полная инструкция: дизайн-система, типы слайдов, готовые компоненты, чек-лист сборки |
| `CLAUDE.md` | Этот файл — что делать при запросе |

## Как реагировать на запрос «собери презентацию»

### Алгоритм

1. **Прочитай оба файла:** `navigator-template.html` и `navigator-guide.md`
2. **Уточни у пользователя** (если не задано):
   - Тема презентации
   - Целевая аудитория
   - Хронометраж видео (для расчёта количества слайдов)
   - Главный вывод / месседж
3. **Скопируй `<style>` блок** из `navigator-template.html` без изменений
4. **Собери структуру слайдов** под тему пользователя:
   - 20–30 слайдов на видео 12–15 мин
   - Группы: открытие (3–4) → контент-блоки (по 2–3 на пункт) → бонус (1–2) → закрытие (3)
5. **Покажи результат** пользователю и пройди по чек-листу ниже

### Жёсткие правила

#### Дизайн (НЕ менять)

**Палитра — только эти цвета:**
- `--parchment` `#F5E6D0` (фон)
- `--terracotta` `#E8762B` (главный акцент)
- `--graphite` `#1E1E1E` (тёмные слайды, плашки)
- `--white` `#FFFFFF`
- `--pain` `#C84B3A` (❌ «что обычно плохо»)
- `--win` `#7A8C3A` (✅ «что должно быть»)
- `--gold` `#B8841A` (инсайт-плашки)

**Никаких** градиентов кроме тёмных слайдов и финальной цитаты. **Никаких** фиолетовых, синих, неоновых, кислотных цветов.

**Шрифты — только эти:**
- `Oswald` 700 — заголовки слайдов и принципов
- `Press Start 2P` — цифры принципов, теги, метки
- `Inter` 400–700 — основной текст, буллиты
- `JetBrains Mono` 400–700 — код, команды, формулировки

#### Структура (СОБЛЮДАЙ)

- Каждый слайд = `<section class="slide">` с `min-height: 100vh` и `scroll-snap-align: start`
- У каждого слайда атрибут `data-slide="N"` с порядковым номером
- В начале каждого слайда — `<div class="slide-tag">` с пиксельной меткой типа `02 · АВТОР` или `КЕЙС · DRIVE-BY REFACTORING`
- Каждая сильная цитата эксперта — отдельный слайд `class="slide dark"` с `karpathy-card`
- Каждый кейс «было/стало» — отдельный слайд с `case-vs` (две колонки `case-side.bad` и `case-side.good`)
- Финальная цитата и главный вывод — на полный экран в самом конце
- Лид-магнит / CTA — последним слайдом с `gift-list` и `gift-cta`

#### Что ЗАПРЕЩЕНО класть в презентацию

- ❌ **Open loops** для удержания зрителя («сейчас покажу...», «через 5 минут...»)
- ❌ **Тайминг съёмки** («1.5 мин», «4 мин» — это для телесуфлёра, не для зрителя)
- ❌ **Метки** «📷 камера / 🖥 экран»
- ❌ **Карточки** «⚠ Адаптировать» с инструкциями ведущему
- ❌ **Внутренние пометки** «не забудь сказать», «здесь переключиться»

Это всё — для шпаргалки ведущего, не для зрителя на трансляции.

#### Технические запреты

- ❌ **Никаких inline-стилей** — только классы. Если нужна разовая стилизация, добавь utility-класс в `<style>`
- ❌ **Не переписывай CSS** из эталона — копируй блок `<style>` целиком
- ❌ **Не сокращай** дизайн-систему — все цвета, шрифты, размеры берутся из эталона как есть
- ❌ **Не ставь** скорость прокрутки на старте больше 1.5 — глаза не успеют

## Готовые компоненты слайдов (бери из эталона)

| Компонент | Когда использовать |
|---|---|
| `hero-num` | Огромная цифра на титульном слайде (43.000 / 65 / etc) |
| `slide-title` + `bullets` | Слайд с 3–5 буллитами с пиксельными цифрами |
| `principle-hero` | Принцип/правило/закон с большой цифрой 01-04 |
| `karpathy-card` (на `slide.dark`) | Цитата эксперта на отдельном слайде |
| `case-vs` | Двухколоночное «было / стало» |
| `terminal` | Терминал с командой, скриптом, промтом |
| `compare-table` | Сравнительная таблица с тёмной шапкой |
| `slide.accent` | Терракотовый слайд для главных выводов |
| `final-quote-card` (на `slide.dark`) | Финальная цитата на полный экран |
| `gift-list` + `gift-cta` | Лид-магнит / CTA в финале |

## Чек-лист перед выдачей

Перед тем как сказать «готово», проверь сам:

- [ ] Прочитал `navigator-template.html` и `navigator-guide.md`
- [ ] Скопирован блок `<style>` из эталона целиком, без правок
- [ ] 20–30 слайдов под хронометраж пользователя
- [ ] У каждого слайда `data-slide` и `slide-tag` сверху
- [ ] Все цитаты экспертов — отдельными слайдами (`slide.dark` + `karpathy-card`)
- [ ] Все кейсы «было/стало» — отдельными двухколоночными слайдами
- [ ] Финальная цитата + главный вывод + лид-магнит на полный экран в конце
- [ ] Никаких inline-стилей в HTML
- [ ] Прогресс-бар, счётчик слайдов, кнопки навигации — из эталона
- [ ] Хоткеи стрелок и пробела работают
- [ ] Открой результат в браузере и прокликай слайды — нет ли пустых, не сломан ли scroll-snap

## Если пользователь говорит «другой стиль / другие цвета»

Объясни:
- Палитра «Крафтовый терминал» — это **бренд-стандарт** проекта. Менять её нельзя без согласования с автором кита
- Если нужен другой стиль — это **другой кит**, не этот
- Если пользователь настаивает — предупреди и попроси явное «да, понимаю что отступаю от стандарта»

## Если что-то непонятно

Лучше задай 1 вопрос, чем сделай неправильно. Точные дыры в брифе:
- Какая тема и аудитория?
- Какой главный вывод видео?
- Сколько по времени видео?
- Есть ли цитаты экспертов? Кейсы? Сравнения?
- Что в финале — лид-магнит, подписка, конкретный CTA?
